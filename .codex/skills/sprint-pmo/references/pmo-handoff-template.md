# PMO Handoff Template

Use this when Codex hands execution to Claude VS Code after a sprint review or architecture decision.

## Handoff Header

- Request name:
- Source sprint:
- Requested by:
- Execution owner: `Claude VS Code`
- Escalation owner: `Human`
- PMO coordinator: `Codex`

## 1. Objective

State the concrete outcome Claude should produce.

## 2. Why This Matters Now

Describe the sprint or architecture reason in two lines or fewer.

## 3. In Scope

- files, modules, or systems allowed to change
- expected user-facing or architectural result

## 4. Out of Scope

- changes that must not be included
- future work intentionally reserved for later

## 5. Execution Notes

- constraints Claude must preserve
- existing patterns to follow
- dependencies or sequencing requirements

## 6. Acceptance Checks

- required tests or validation
- observable completion criteria
- documentation or status updates needed

## 7. Escalate to Human If

- architecture direction is unclear
- a cross-lane tradeoff appears
- delivery requires expanding scope
- a risk changes sprint sequencing

## 8. PMO Closeout Format

Ask Claude to return:

- what changed
- what remains
- what needs human review
- recommended next sprint slice
