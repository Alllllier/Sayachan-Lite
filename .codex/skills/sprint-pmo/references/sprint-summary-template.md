# Sprint Summary Template

Use this when closing a sprint or replying to PMO.

## Header

- Sprint: `SXX - short name`
- Date window: `YYYY-MM-DD to YYYY-MM-DD`
- PMO owner: `Codex`
- Execution worker: `Claude VS Code`
- Architecture owner: `Human`
- Program lane: `AI core | product runtime | future labs | mixed`

## 1. Sprint Goal

State the intended sprint outcome in one or two lines.

## 2. Completion Status

- `Complete`: list only items with evidence
- `Complete, pending validation`: list work finished but not fully verified
- `In progress`: list partially delivered items
- `Deferred`: list items intentionally moved out
- `Dropped`: list items explicitly removed from scope

## 3. Delivery Delta

- Planned versus delivered: note the largest delta in one short paragraph
- Why it changed: capture the real reason, not a generic blocker label

## 4. Architecture Notes

- Decisions made
- Decisions still needed from the human
- Constraints introduced for the next sprint

## 5. Risk and Debt Snapshot

List only material items. Use one line each:

- `[Risk|Debt]` area: impact, owner, next action

Use the checklist in `risk-debt-checklist.md` if a deeper pass is needed.

## 6. PMO Reply

Write a short PMO-facing reply:

`Sprint closed with <result>. Main completion: <x>. Remaining gap: <y>. Proposed next sprint: <z>.`
