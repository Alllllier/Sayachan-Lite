# AI Ops

This directory stores the repository's AI-assisted development operating system.

Use it to maintain the long-lived rules, workflows, and reference assets that govern:

- Codex + Claude + Human collaboration
- PMO handoff and execution loops
- documentation ownership and sync expectations
- skill growth and policy updates
- cost-aware model routing

## Directory Map

- `architecture/` - structure and design principles for the AI development system itself
- `workflows/` - operational loops for planning, execution, review, and maintenance
- `policies/` - updateable rules that skills and PMO flows should reference
- `history/` - migration records for major process changes

Important references:

- `architecture/ai-development-system.md`
- `architecture/public-private-development-model.md`

## Maintenance Rule

Update this directory when one of these changes:

- agent role split
- repo-native handoff shape
- PMO state model
- documentation ownership boundaries
- model-routing rules
- skill growth rules

Do not update it for every sprint. Treat it as operating-system documentation, not sprint output.
