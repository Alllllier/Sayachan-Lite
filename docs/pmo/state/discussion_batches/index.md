# Discussion Batches

> Top-level index for clustered discussion batches before they enter the formal PMO pipeline.

## Usage Rule

- each batch represents one discussion wave from the human
- each batch should point to one canonical batch file
- each batch may contain multiple clustered themes
- only one theme per batch should be `in_focus` at a time
- a theme may also record lightweight `possible slices` when one discussion topic contains multiple likely execution cuts
- promote items into `idea_backlog.md`, `sprint_candidates.md`, or `decision_log.md` only after the relevant theme is stable
