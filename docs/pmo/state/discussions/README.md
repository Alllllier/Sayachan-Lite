# Discussions

This directory stores discussion batches.

Each batch should:

- preserve clustered discussion themes
- preserve execution-relevant slice granularity
- make promotion outcomes explicit
- declare whether it is `exploration` or `follow-up`

Discussion files are not the long-term canonical home for stable planning results.

Once a result is stable, promote it into:

- `idea_backlog.md`
- `sprint_candidates.md`
- `decision_log.md`

`exploration` batches are best for broad pre-sprint direction shaping.
`follow-up` batches are best for post-closeout, post-execution, or real-usage single-topic discussion.
