- Delivered summary:
  - Added phase-one backend auth records and services:
    - `backend/src/models/User.js`
    - `backend/src/models/Invite.js`
    - `backend/src/models/Session.js`
    - `backend/src/services/authService.js`
    - `backend/src/middleware/auth.js`
    - `backend/src/routes/authRoutes.js`
  - Added secure password hashing using Node `crypto.pbkdf2Sync` with per-user salts; no password dependency was added.
  - Added cookie-backed session login/logout/current-user behavior using the `sayachan_session` httpOnly cookie.
  - Added owner bootstrap at `POST /auth/bootstrap-owner`.
    - Bootstrap creates the first owner only when no owner exists.
    - Bootstrap assigns legacy pre-auth Notes, Projects, and Tasks with missing `userId` to the owner.
  - Added tester registration at `POST /auth/register`.
    - Requires email, password, and invite code.
    - Invite codes are not email-bound.
    - Invite codes are single-use, expiring, and revocable.
  - Added owner-only routes:
    - `POST /owner/invites`
    - `GET /owner/invites`
    - `POST /owner/invites/:id/revoke`
    - `GET /owner/testers`
    - `POST /owner/testers/:id/disable`
    - `POST /owner/testers/:id/restore`
    - `GET /owner/system-status`
  - Added minimal auth-skeleton data safety for product data:
    - `userId` fields on Note, Project, and Task.
    - Notes, Projects, and Tasks are tagged with the current user on create.
    - Notes, Projects, and Tasks list/update/delete/archive/restore/pin/unpin operations are filtered by the current user when routed through authenticated requests.
    - This was limited to the minimum needed for tester accounts to start empty and for legacy data to belong to the owner.
  - Added frontend auth shell:
    - `frontend/src/views/LoginPage.vue`
    - `frontend/src/views/RegisterPage.vue`
    - `frontend/src/views/OwnerPage.vue`
    - `frontend/src/stores/auth.js`
    - `frontend/src/features/auth/auth.api.js`
    - `frontend/src/services/apiClient.js`
  - Added route guards in `frontend/src/router/index.js`.
    - Normal app routes require authentication.
    - Owner route requires owner role.
    - Public auth screens redirect authenticated users back to the app.
  - Updated `frontend/src/App.vue` to expose current-user shell, logout, owner navigation for owners only, and to hide app navigation/chat on auth screens.
  - Updated existing frontend API boundaries to send credentials so the Vite frontend can use backend session cookies.

- Residual risks or escalations:
  - The first-owner bootstrap endpoint is intentionally public until an owner exists; production use should pair this with deployment/operator controls.
  - The minimal `userId` scoping covers Notes, Projects, and Tasks product routes only. It does not claim to complete the broader account isolation candidate.
  - Current owner invite creation returns the full invite code only at creation time; later listing returns only preview metadata.
  - Disabled testers lose stored sessions through `Session.deleteMany` and are also rejected on the next authenticated request; this depends on the request path passing through the new auth middleware.
