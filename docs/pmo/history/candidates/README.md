# Candidate History

这一层用于承载已经退出 `state/sprint_candidates.md` 的 completed candidate 记录。

它回答的是：

- 最近或过去有哪些 sprint candidate 已经完成并退出当前候选面
- 某个 follow-up sprint 是从哪一个已完成 candidate 长出来的
- 为什么某个已完成 slice 不再属于当前候选比较面

这里不承担当前 PMO 的 active candidate 比较，也不替代：

- `state/current_sprint.md` 的最近 closeout 记录
- `state/sprint_candidates.md` 的当前候选面

使用规则：

- 当 `state/sprint_candidates.md` 因总量限制需要移出较早的 completed entry 时，将它归档到这里
- 归档时参照 `candidate-archive.template.md`
- 不要把 still-open candidate 放到这里
