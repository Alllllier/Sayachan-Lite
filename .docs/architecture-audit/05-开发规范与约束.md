## 1. 代码风格规范

### 1.1 Vue 前端代码风格

#### 组件结构
```vue

// 2. 常量定义
const API_BASE = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001'

// 3. 响应式状态（按功能分组）
// 数据状态
const items = ref([])
const form = ref({ title: '', content: '' })

// UI 状态
const loading = ref(false)
const error = ref(null)
const isEditing = ref(false)

// 4. 计算属性
const safeItems = computed(() => Array.isArray(items.value) ? items.value : [])

// 5. 方法定义
async function fetchData() {
  // ...
}

// 6. 生命周期
onMounted(fetchData)
</script>

<template>
  <!-- 模板内容 -->
</template>

<style scoped>
/* 样式 */
</style>
```

#### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `NotesPanel.vue`, `EmptyState.vue` |
| 组件引用 | PascalCase | `<NotesPanel />` |
| 变量/函数 | camelCase | `fetchNotes`, `isLoading` |
| 常量 | UPPER_SNAKE_CASE | `API_BASE` |
| 布尔变量 | is/has/show 前缀 | `isPinned`, `hasError`, `showArchived` |
| 事件处理 | handle 前缀 | `handleSubmit`, `handleTaskComplete` |
| CSS 类名 | kebab-case | `.card-buttons`, `.form-section` |
| 自定义事件 | 动词过去式 | `@refreshed`, `@updated` |

#### 响应式状态组织顺序
1. 数据状态（items, formData）
2. UI 状态（loading, error, editingId）
3. 计算属性（computed）
4. 方法（functions）
5. 生命周期钩子

#### 错误处理模式
```javascript
async function someOperation() {
  loading.value = true
  error.value = null
  try {
    const res = await fetch(`${API_BASE}/endpoint`)
    if (!res.ok) throw new Error('Operation failed')
    const data = await res.json()
    // 成功处理
  } catch (e) {
    console.error('Failed to operation:', e)
    error.value = 'User-friendly error message'
    showToast('Error message', 'error')
  } finally {
    loading.value = false
  }
}
```

#### 成功提示模式
```javascript
successMessage.value = 'Operation succeeded'
setTimeout(() => { successMessage.value = '' }, 2000)
```

### 1.3 CSS 风格规范

#### 类名命名
- 使用 kebab-case
- 语义化命名，避免具体样式值
- 使用 BEM 变体（Block-Element-Modifier）

```css
/* Block */
.card { }

/* Element */
.card-title { }
.card-content { }

/* Modifier */
.card.archived { }
.card-accent-green { }

/* State */
.is-active { }
.is-loading { }
```

#### 样式优先级
- 基础样式在全局（App.vue 或 main.css）
- 组件特定样式使用 `<style scoped>`
- 避免 `!important`，除非覆盖第三方库

#### 颜色值（优先使用 CSS 变量）
- 主色：`var(--action-primary)`（#42b883）
- 错误：`var(--status-error)`（#ef4444）
- 成功：`var(--status-success)`（#10b981）
- 警告：`var(--status-warning)`（#f59e0b）
- 中性色：`var(--text-primary)`, `var(--text-secondary)`, `var(--text-muted)`

#### 基线 CSS 使用规范（重要）

**优先使用 `style.css` 中定义的基线类，避免重复定义**

基线 CSS 文件：`frontend/src/style.css`

**基线类清单**:

| 类别 | 基线类名 | 用途 | 禁止自定义的样式 |
|------|----------|------|------------------|
| **按钮** | `.btn` | 基础按钮 | padding、border、transition、基础颜色 |
| | `.btn-primary` | 主按钮（绿） | background、color |
| | `.btn-secondary` | 次按钮（靛蓝） | background、color |
| | `.btn-danger` | 危险按钮（红） | background、color |
| | `.btn-archive` | 归档按钮（灰蓝） | background、color |
| | `.btn-ai` | AI 按钮（金） | background、color |
| | `.btn-sm` / `.btn-lg` | 尺寸变体 | padding、font-size |
| **输入框** | `.input` | 单行输入 | padding、border、focus 样式 |
| | `.textarea` | 多行输入 | padding、border、focus 样式 |
| **卡片** | `.card` | 基础卡片 | padding、border、radius、shadow |
| | `.card-elevated` | 带阴影卡片 | box-shadow |
| | `.card-interactive` | 可交互卡片 | hover 效果 |
| | `.card-accent-*` | 强调边框 | border-left |

**正确示例**:
```vue
<template>
  <!-- 使用基线按钮 -->
  <button class="btn btn-primary">Save</button>
  <button class="btn btn-secondary btn-sm">Edit</button>
  
  <!-- 使用基线卡片 -->
  <div class="card card-accent-green">
    <h3 class="card-title">Title</h3>
    <p class="card-content">Content</p>
  </div>
  
  <!-- 使用基线输入框 -->
  <input class="input" v-model="title" />
  <textarea class="textarea" v-model="content"></textarea>
</template>

<style scoped>
/* 只写组件特有样式 */
.my-component {
  display: flex;
  gap: var(--space-md);
}

.my-component .custom-element {
  color: var(--text-secondary);
}
</style>
```

**错误示例**（避免）:
```vue
<style scoped>
/* ❌ 重复定义按钮基础样式 */
.btn {
  padding: 10px 20px;  /* style.css 已有 */
  border: none;         /* style.css 已有 */
  transition: 0.2s;     /* style.css 已有 */
}

/* ❌ 硬编码颜色 */
.custom-btn {
  background: #42b883;  /* 应使用 var(--action-primary) */
}

/* ❌ 重复定义输入框 focus */
.input:focus {
  outline: none;        /* style.css 已有 */
  border-color: #42b883; /* style.css 已有 */
}
</style>
```

**例外情况**（可以自定义）:
- 组件特有布局（flex、grid、position）
- 组件特有尺寸（width、height）
- 组件特有动画
- 基线未覆盖的特殊样式

**基线未覆盖时的处理**:
如果需要的样式不在基线中，先在 `style.css` 中添加，再在组件中使用。

## 2. API 接口模板

### 2.1 新增 CRUD 接口模板

#### 后端路由模板
```javascript
const Router = require('@koa/router')
const Model = require('../models/Model')

const router = new Router()

// GET /resources - 列表（支持 archived 筛选）
router.get('/resources', async (ctx) => {
  const { archived } = ctx.query
  const filter = archived === 'true'
    ? { status: 'archived' }
    : { status: { $ne: 'archived' } }
  
  const items = await Model.find(filter)
    .sort({ isPinned: -1, pinnedAt: -1, updatedAt: -1 })
  
  ctx.body = items
})

// GET /resources/:id - 单个（如需要）
router.get('/resources/:id', async (ctx) => {
  const item = await Model.findById(ctx.params.id)
  if (!item) {
    ctx.status = 404
    ctx.body = { error: 'Resource not found' }
    return
  }
  ctx.body = item
})

// POST /resources - 创建
router.post('/resources', async (ctx) => {
  const body = ctx.request.body
  const item = await Model.create({
    field1: body.field1,
    field2: body.field2 || 'default'
  })
  ctx.status = 201
  ctx.body = item
})

// PUT /resources/:id - 更新
router.put('/resources/:id', async (ctx) => {
  const id = ctx.params.id
  const body = ctx.request.body
  const item = await Model.findByIdAndUpdate(
    id,
    { field1: body.field1, field2: body.field2 },
    { new: true, runValidators: true }
  )
  if (!item) {
    ctx.status = 404
    ctx.body = { error: 'Resource not found' }
  } else {
    ctx.body = item
  }
})

// DELETE /resources/:id - 删除
router.delete('/resources/:id', async (ctx) => {
  const item = await Model.findByIdAndDelete(ctx.params.id)
  if (!item) {
    ctx.status = 404
    ctx.body = { error: 'Resource not found' }
  } else {
    ctx.status = 204
    ctx.body = null
  }
})

// PUT /resources/:id/archive - 归档（如需要）
router.put('/resources/:id/archive', async (ctx) => {
  const item = await Model.findByIdAndUpdate(
    ctx.params.id,
    { status: 'archived' },
    { new: true }
  )
  if (!item) {
    ctx.status = 404
    ctx.body = { error: 'Resource not found' }
    return
  }
  console.log(`[Resource Archive] "${item.name}" archived`)
  ctx.body = item
})

module.exports = router
```

#### 前端 Service 模板
```javascript
// services/resourceService.js
import { ref } from 'vue'

const API_BASE = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001'

export const resourcesRef = ref([])

export async function fetchResources(archived = false) {
  try {
    const url = archived 
      ? `${API_BASE}/resources?archived=true` 
      : `${API_BASE}/resources`
    const res = await fetch(url)
    const data = await res.json()
    resourcesRef.value = data
    return resourcesRef.value
  } catch (e) {
    console.error('Failed to fetch resources:', e)
    return []
  }
}

export async function createResource(data) {
  try {
    const res = await fetch(`${API_BASE}/resources`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    })
    const newItem = await res.json()
    if (newItem) {
      resourcesRef.value.unshift(newItem)
    }
    return newItem
  } catch (e) {
    console.error('Failed to create resource:', e)
    return null
  }
}

export async function updateResource(id, data) {
  try {
    const res = await fetch(`${API_BASE}/resources/${id}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    })
    const updated = await res.json()
    const index = resourcesRef.value.findIndex(r => r._id === id)
    if (index !== -1) {
      resourcesRef.value[index] = updated
    }
    return updated
  } catch (e) {
    console.error('Failed to update resource:', e)
    return null
  }
}

export async function deleteResource(id) {
  try {
    await fetch(`${API_BASE}/resources/${id}`, { method: 'DELETE' })
    resourcesRef.value = resourcesRef.value.filter(r => r._id !== id)
    return true
  } catch (e) {
    console.error('Failed to delete resource:', e)
    return false
  }
}
```

## 3. Git Commit Message 规范

### 3.1 格式
```
<type>(<scope>): <subject>

[可选的 body]

[可选的 footer]
```

### 3.2 Type（必填）

| Type | 含义 | 使用场景 |
|------|------|----------|
| `feat` | 新功能 | 新增功能、模块、页面 |
| `fix` | 修复 | 修复 bug |
| `refactor` | 重构 | 代码重构，不改变功能 |
| `style` | 样式 | UI 调整、CSS 修改 |
| `docs` | 文档 | 文档更新 |
| `milestone` | 里程碑 | 重大版本节点 |
| `remove` | 删除 | 删除文件或功能 |

### 3.3 Scope（可选）

| Scope | 含义 |
|-------|------|
| `ui` | 用户界面 |
| `workbench` | 工作台整体 |
| `cockpit` | Dashboard |
| `project` | 项目模块 |
| `task` | 任务模块 |
| `note` | 笔记模块 |
| `navigation` | 导航 |
| `api` | 后端接口 |
| `beta` | Beta 测试相关 |
| `backend` | 后端整体 |

### 3.4 Subject（必填）

- 使用祈使句，现在时（"add" 而非 "added"）
- 首字母小写
- 不超过 50 个字符
- 不加句号

### 3.5 示例

```bash
# 功能新增
feat(project): add archive functionality with cascade to tasks
feat(note): add pin/unpin feature

# Bug 修复
fix(ui): close interaction precision patches for AI suggestion flow
fix(beta): validate focus transition logic

# 重构
refactor(task): introduce semantic task fields
refactor(workbench): consolidate task semantics

# 样式调整
style(ui): optimize edit mode button placement
style(ui): add vertical spacing to note card action buttons

# 里程碑
milestone: rename to Sayachan Lite and prepare deploy-ready version
milestone: deploy-ready v0.1

# 删除
remove vercel.json
```

### 3.6 提交频率建议

- 保持提交小而集中
- 一个 commit 完成一个逻辑单元
- 避免将不相关的改动混在一起

## 4. 现有风格不一致记录

### 4.1 已发现的不一致

| 位置 | 问题 | 现状 | 建议统一为 |
|------|------|------|-----------|
| `backend/src/routes/index.js` | 日志格式 | `[Focus Transition]` 有空格 | `[FocusTransition]` 无空格 |
| `NotesPanel.vue` | 注释风格 | `// P0-Fix-1:` 带连字符 | `// P0 Fix 1:` 或移除 |
| `Dashboard.vue` | 注释风格 | `// P0-1:`, `// P0-2:`, `// P0-B:` | 统一注释格式或移除 |
| `Task.js` | 字段命名 | legacy 字段与 semantic 字段并存 | 保留兼容，新代码只用 semantic |
| `NotesPanel.vue` | 错误提示 | 部分用 toast，部分用 error.value | 统一使用 toast |
| 全局 | 魔法数字 | 多处使用 `3000`, `2000` 等超时 | 提取为常量或保持现状 |

### 4.2 允许存在的差异

| 差异 | 原因 | 说明 |
|------|------|------|
| 前端/后端引号风格 | 历史原因 | 前端单引号，后端混用，不强求统一 |
| 空行数量 | 可读性 | 函数间 1-2 空行均可接受 |
| 注释详细程度 | 上下文不同 | 复杂逻辑多注释，简单逻辑可省略 |

## 5. 开发约束

### 5.1 明确不做

| 约束 | 说明 |
|------|------|
| ❌ TypeScript | 保持 JavaScript，减少构建复杂度 |
| ❌ UI 组件库 | Element Plus / Ant Design 等不引入 |
| ❌ 复杂路由改造 | 当前路由结构满足需求 |
| ❌ 大型功能系统 | 保持轻量，不增加新的大模块 |
| ❌ 架构升级重构 | 如无必要勿增实体 |

### 5.2 状态管理约定

**业务数据**：继续使用简单的 `ref` 导出

```javascript
// 推荐：简单的 ref 导出
// services/taskService.js
import { ref } from 'vue'

export const tasksRef = ref([])

export async function fetchTasks() {
  // 更新 tasksRef.value
}

// 在组件中使用
import { tasksRef, fetchTasks } from '../services/taskService'
// 直接使用 tasksRef.value
```

**全局跨路由状态**（如 Chat、Cockpit signals）：使用 Pinia setup store

```javascript
// stores/chat.js
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useChatStore = defineStore('chat', () => {
  const isOpen = ref(false)
  const messages = ref([])
  // ...
  return { isOpen, messages }
})
```

### 5.3 新增字段约定

- 模型新增字段必须有默认值
- 保持向后兼容
- 避免删除已有字段（标记 deprecated 即可）

### 5.4 AI 功能开发约定

- 所有 AI 功能必须有 fallback
- AI 调用优先走后端代理（安全性）
- Dashboard AI 可保留前端直连（可选迁移）
- Prompt 必须使用简体中文
