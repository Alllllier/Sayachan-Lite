# Sayachan Lite - 技术栈

> 项目架构审计文档 | 最后更新: 2026-04-15

---

## 1. 整体架构

Sayachan Lite 采用经典的前后端分离架构：

```
┌─────────────────┐     HTTP/REST     ┌─────────────────┐     ┌─────────────────┐
│     Frontend    │ ◄───────────────► │    Backend      │ ◄──►│   MongoDB       │
│   (Vue 3 SPA)   │    (JSON API)     │   (Koa.js)      │     │  (Mongoose)     │
└─────────────────┘                   └─────────────────┘     └─────────────────┘
       │                                       │
       ▼                                       ▼
  Vite Dev Server                      Node.js Runtime
  (localhost:5173)                     (localhost:3001)
```

## 2. 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Vue** | 3.5.30+ | 核心框架 (Composition API) |
| **Vue Router** | 4.6.4+ | 客户端路由 |
| **Pinia** | 3.0.4+ | 全局状态管理 (Chat / Cockpit signals) |

| **@vitejs/plugin-vue** | 6.0.5+ | Vue SFC 支持 |

### 2.2 前端设计决策

- **状态管理**: Chat 与跨路由状态使用 Pinia；业务数据继续使用简单的 `ref` 导出
- **无 UI 组件库**: 原生 CSS 实现，保持极简
- **无 TypeScript**: JavaScript 原生开发
- **Service 模式**: API 调用封装在 services/ 目录

```
│   │   ├── adaptor/     # Provider 适配层 (Kimi)
│   │   ├── personality/ # 人格 Prompt 层
│   │   ├── context/     # 会话上下文层
│   │   ├── orchestration/  # 编排服务层

│   └── server.js        # 应用入口

└── .env                 # 环境变量 (gitignored)
```

### 3.2 后端设计决策

- **非阻塞数据库启动**: MongoDB 连接失败不会阻止服务启动
- **模块化路由**: 业务路由与 AI 路由分离
- **统一错误处理**: 404 统一处理中间件
- **环境驱动配置**: 通过环境变量控制所有可变配置

## 4. 数据库

| 技术 | 用途 |
|------|------|
| **MongoDB** | 主数据库 |
| **Mongoose** | 对象建模 + 验证 |

### 4.1 数据模型概览

```javascript
// Note 模型
{
  title: String,        // required
  content: String,      // default: ''
  status: Enum['active', 'archived'],  // default: 'active'
  isPinned: Boolean,    // default: false
  pinnedAt: Date,       // default: null
  timestamps: true      // createdAt, updatedAt
}

// Project 模型
{
  name: String,         // required
  summary: String,      // required
  status: Enum['pending', 'in_progress', 'completed', 'on_hold', 'archived'],
  nextAction: String,   // 当前聚焦点
  lastCompletedAction: String,  // 最近完成项 (deprecated)
  focusHistory: [String],       // 推进轨迹数组
  isPinned: Boolean,
  pinnedAt: Date,
  timestamps: true
}

// Task 模型
{
  title: String,        // required
  // 语义化字段
  creationMode: Enum['ai', 'manual'],
  originModule: String, // 'note' | 'project' | 'dashboard' | 'project_focus' | etc.
  originId: Mixed,      // 来源ID
  originLabel: String,  // 来源标签
  linkedProjectId: ObjectId,   // 关联项目ID
  linkedProjectName: String,   // 关联项目名称
  // 遗留字段 (兼容性)
  source: String,
  sourceDetail: String,
  projectId: ObjectId,
  projectName: String,
  // 状态
  status: Enum['active', 'completed', 'archived'],
  completed: Boolean,
  timestamps: true
}
```

## 5. AI 集成

| 服务 | 模型 | 用途 |
|------|------|------|
| **智谱 GLM** | glm-4.5-air | AI 文本生成 (Notes/Projects) |
| **Kimi** | kimi-k2.5 | AI 文本生成 (Chat) |

### 5.1 AI 调用方式

| 功能 | 调用位置 | 说明 |
|------|----------|------|
| 笔记生成任务 | 后端代理 (GLM) | `/ai/notes/tasks` |
| 项目下一步建议 | 后端代理 (GLM) | `/ai/projects/next-action` |
| 通用对话聊天 | 后端代理 (Kimi) | `/ai/chat` |
| 文本摘要 | 前端直连 | `aiService.summarizeText()` |
| 周回顾 | 前端直连 | `aiService.generateWeeklyReview()` |
| 聚焦推荐 | 前端直连 | `aiService.recommendFocus()` |
| 行动计划 | 前端直连 | `aiService.generateActionPlan()` |

### 5.2 AI 配置

```bash
# 后端 (必需用于 Notes/Projects AI)
GLM_API_KEY=your_api_key_here

# 前端 (Dashboard AI 需要)
VITE_GLM_API_KEY=your_api_key_here  # 部分功能已迁移至后端
```

## 6. 部署架构

### 6.1 推荐生产部署

| 组件 | 平台 | 说明 |
|------|------|------|
| Frontend | **Vercel** | 静态托管，自动构建 |
| Backend | **Render** | Node.js Web Service |
| Database | **MongoDB Atlas** | 云端 MongoDB |

### 6.2 环境变量配置

**Vercel (Frontend)**:
```
VITE_API_BASE_URL=https://your-backend.onrender.com
```

**Render (Backend)**:
```
MONGO_URI=mongodb+srv://...
FRONTEND_ORIGIN=https://your-app.vercel.app
PORT=10000  # Render 默认
GLM_API_KEY=...
```

- ❌ 大型功能系统新增
- ❌ 架构升级式重构
