# Sayachan Lite - 文档同步指南

> 当 architecture 相关代码改动时，如何同步更新文档

---

## 1. 自动检查机制

### 1.1 Git Pre-commit Hook

项目配置了 pre-commit hook，当提交 architecture 相关文件时会自动提醒。

**安装方式**:
```bash
bash .claude/hooks/install-hooks.sh
```

**触发条件**:
- 修改了 `backend/src/models/*.js`
- 修改了 `backend/src/routes/*.js`
- 修改了 `frontend/src/services/*.js`
- 修改了 `frontend/src/router/*.js`

**Hook 行为**:
- 如果修改了 architecture 文件但未更新文档，提交会被阻止
- 提示需要检查哪些文档
- 使用 `git commit --no-verify` 可跳过（不推荐）

## 3. 文档更新流程

### 3.1 标准流程

```
修改代码 → 测试功能 → 更新文档 → 一起提交
```

### 3.2 具体步骤

**Step 1: 修改代码**
```bash
# 修改 architecture 相关文件
vim backend/src/models/Note.js
vim backend/src/routes/index.js
```

**Step 2: 本地测试**
```bash
# 确保功能正常
npm run dev
npm start
```

**Step 3: 更新对应文档**
根据上面的"变更类型与文档对应表"，更新相关文档。

**Step 4: 一起提交**
```bash
git add backend/src/models/Note.js
git add .docs/architecture-audit/03-后端API接口.md
git commit -m "feat(note): add newField to Note model"
```

## 5. 快速检查清单

### 5.1 提交前检查

修改以下文件后，请检查：

**backend/src/models/*.js**
- [ ] `03-后端API接口.md` 的 Schema 章节已更新
- [ ] `01-技术栈.md` 的数据模型概览已更新（如必要）

**backend/src/routes/*.js**
- [ ] `03-后端API接口.md` 的接口总览表已更新
- [ ] 详细接口说明已添加/更新
- [ ] 请求/响应示例已更新

**frontend/src/services/*.js**
- [ ] `02-核心能力与workflow.md` 的数据流转图已更新（如必要）
- [ ] `01-技术栈.md` 的关键文件索引已更新（如新增文件）

**frontend/src/router/*.js**
- [ ] `01-技术栈.md` 的前端项目结构已更新

**frontend/src/style.css**
- [ ] `05-开发规范与约束.md` 的基线 CSS 使用规范已更新（如新增基线类）
- [ ] 所有使用新基线类的组件已更新

### 5.2 Roadmap 更新时机

以下情况需要更新 `04-Roadmap.md`：
- [ ] 新增功能完成
- [ ] 重要 Bug 修复
- [ ] 性能优化
- [ ] 架构调整

## 7. 忽略文档更新的场景

以下情况可以跳过文档更新：

1. **纯代码重构** - 不改变接口契约、数据模型的重构
2. **Bug 修复** - 修复实现问题，不改变设计
3. **样式调整** - 纯 UI/CSS 改动
4. **日志/注释** - 仅修改日志或注释

**注意**: 如果不确定是否需要更新，建议更新。

## 8. 自动化检查脚本（可选）

可以创建脚本定期检查文档同步情况：

```bash
#!/bin/bash
# check-doc-sync.sh

echo "Checking documentation sync..."

# 获取最近修改的 architecture 文件
ARCH_FILES=$(git log --since="1 week ago" --name-only --pretty=format: | \
    grep -E "backend/src/(models|routes)/.*\.js|frontend/src/(services|router)/.*\.js" | \
    sort | uniq)

# 获取最近修改的文档文件
DOC_FILES=$(git log --since="1 week ago" --name-only --pretty=format: | \
    grep -E "\.docs/architecture-audit/.*\.md|CLAUDE\.md" | \
    sort | uniq)

if [ -n "$ARCH_FILES" ] && [ -z "$DOC_FILES" ]; then
    echo "⚠️  Architecture files modified recently but no documentation updates:"
    echo "$ARCH_FILES"
    exit 1
fi

echo "✓ Documentation sync check passed"
```
